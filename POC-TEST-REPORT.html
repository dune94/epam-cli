EPAM CLI POC

Governed AI Delivery CLI Proven Through End-to-End Operator Testing

This proof of concept validated EPAM CLI as a working control-plane product: authenticated execution, proxy-backed AI runs, approval-gated delivery phases, cost reporting, delivery memory, and Linux-safe credential fallback were all exercised with captured evidence.

Outcome

  • Core CLI path is operational against a mock EPAM backend.
  • Governed delivery flow is working, including explicit phase approval and audit logging.
  • Key operator defects discovered during testing were fixed in the product, not just documented.
  • Remaining issues are narrowed to low-priority runtime UX warnings now tracked in backlog.

Environment

  • Workspace build from `/home/bjerome/projects/ai/epam-cli`
  • Mock EPAM backend at `http://127.0.0.1:8080`
  • No real EPAM SSO integration yet
  • No live Anthropic/OpenAI browser-login integration yet

Validated Product Capabilities

EPAM Auth and Control Plane

`doctor`, `login`, `whoami`, backend health, and provider readiness checks all functioned against the mock backend.

Proxy-Based AI Execution

`run` and `chat` worked through the EPAM provider path after proxy routing was corrected to resolve real upstream targets.

Governed Delivery Phases

`phase approve` and `phase run --require-approval` validated contract-style control over execution with an audit trail.

Delivery Planning

`estimate` and `orchestrate --dry-run` showed both planning-only and AI-assisted forecasting behavior.

Delivery Memory

`decision` and `profile` flows confirmed that reusable architectural guidance and operator profiles can be persisted and loaded.

FinOps and Reporting

`history`, `report`, and `/cost` validated session visibility, cost telemetry, and the current budget UX.

Issues Found and Closed During POC

  • `doctor` misdiagnosis: now identifies placeholder backend URLs as the real issue.
  • EPAM provider proxy route: now resolves to valid upstream provider routes.
  • Report math: `Avg Tokens/Turn` no longer returns `NaN`.
  • Linux key storage: `keys` now falls back when DBus secret-service is unavailable.
  • Pricing gap: `/cost` now supports `claude-sonnet-4-5-20250929`.
  • Completed phase UX: `phase run` now fails clearly when no runnable stories exist.

Remaining Backlog Bugs

  • EPAM-049: disable dead default MCP example server warning.
  • EPAM-050: remove misleading provider-slot warning during successful EPAM REPL startup.

Both are intentionally tracked as low-priority because they do not block governed execution, planning, or reporting.
