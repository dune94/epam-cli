## KB-008 -- 2026-06-01

**Category:** backend
**AgentRole:** typescript
**Tags:** typescript, mcp-protocol, cli-ux
**Trigger:** first-success
**StoryRef:** EPAM-049

When shipping example MCP server configurations, mark them with `enabled: false` by default to avoid startup noise. The McpClient initialization filter (`s.enabled !== false`) treats undefined/missing `enabled` field as enabled (default true), allowing user-added servers to work without modification while disabled examples stay silent. Add an optional `enabled` field to McpServerConfig type and filter servers during loadConfig phase—before attempting connection. This preserves backward compatibility (existing configs without the field work as-is) while cleanly distinguishing documentation examples from active runtime configuration.
