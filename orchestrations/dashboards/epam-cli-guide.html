EPAM CLI User Guide

EPAM CLI is a multi-LLM AI coding assistant with enterprise features: provider failover chains, OAuth 2.0 auth, BYOK API keys, live cost tracking, session persistence, and a full multi-agent orchestration system for executing development phases against a prd.json project plan.

What is EPAM CLI?

EPAM CLI is a TypeScript/Node.js CLI that wraps multiple LLM providers (Anthropic Claude, OpenAI, Gemini) behind a unified interface. It operates in two modes:

  • Interactive REPL (chat) — streamed conversation with slash commands, session history, and tool use
  • Non-interactive agent (run) — single-shot task execution from CLI args or stdin

On top of the agent core sits the orchestration system: a suite of shell scripts and a Node.js inference layer that plans, estimates, and executes multi-story development phases using autonomous Claude Code agents.

Quick Start

# Build (Node 20 required)
~/.nvm/versions/node/v20.20.0/bin/node ./node_modules/.bin/tsup
node dist/epam.js --version

# Start interactive chat
EPAM_API_KEY_ANTHROPIC=sk-ant-... node dist/epam.js chat

# Run a single task non-interactively
echo "List all TypeScript files" | node dist/epam.js run

# Or with a direct prompt
node dist/epam.js run "Write a fibonacci function in TypeScript"

# With a specific model
node dist/epam.js chat --model claude-opus-4-6

# Pipe mode — useful for scripting
cat myfile.ts | node dist/epam.js run "Explain this code"

Note: If running with EPAM proxy (pro/enterprise tier), log in first with node dist/epam.js login. For BYOK (free tier), set EPAM_API_KEY_ANTHROPIC, EPAM_API_KEY_OPENAI, or EPAM_API_KEY_GEMINI environment variables.

Authentication

EPAM CLI supports two authentication modes, automatically selected by tier:

TierAuth ModeHow to authenticate
free BYOK — direct API calls with your own key Set EPAM_API_KEY_* env var or use epam keys set
pro / enterprise OAuth 2.0 + EPAM proxy — keys managed by the backend Run epam login (device flow, opens browser)

Device Flow (pro/enterprise)

epam login
# Opens: https://your-epam-backend/auth/device
# Polls for authorization, then stores token in system keychain (or encrypted file fallback)

epam whoami          # Confirm logged-in identity
epam logout          # Revoke token and clear stored credentials

BYOK Keys

epam keys set anthropic sk-ant-...    # Store key in system keychain
epam keys set openai   sk-...
epam keys set gemini   AIza...

epam keys list                          # Show which providers have stored keys
epam keys get anthropic                 # Show masked key (first 8 + last 4 chars)
epam keys remove anthropic              # Delete stored key

CLI Commands

All commands: node dist/epam.js <command> [options]

epam chat Start an interactive REPL session CORE 
epam chat [--model <model>] [--provider <provider>]

# Examples
epam chat
epam chat --model claude-opus-4-6
epam chat --provider openai --model gpt-4o

Starts the interactive REPL with streaming output, slash commands, tool use, and session persistence. In pipe mode (non-TTY), behaves identically to epam run.

Tools available: ReadFile (safe), WriteFile (review gate), Bash (dangerous — approval required), ListFiles (safe), Search (safe), FetchUrl (safe).

epam run Execute a single agent task non-interactively CORE 
epam run [prompt] [--model <model>] [--provider <provider>] [--no-tools] [--json]

# Read prompt from argument
epam run "Refactor AuthManager to use dependency injection"

# Read from stdin
echo "List files" | epam run
cat spec.md | epam run "Implement this spec"

# JSON output mode (suppresses streaming text, emits structured result)
epam run "Count lines in src/" --json

Runs a single ReAct agent loop, streaming output to stdout. Uses - or piped stdin for the prompt. --json suppresses text streaming and outputs a structured result object.

epam login Authenticate via OAuth 2.0 device flow AUTH 
epam login            # Opens browser for device flow authentication
epam logout           # Revoke token and clear credentials
epam whoami           # Show current identity (email, tier, token expiry)

Uses RFC 8628 device flow — no local callback server needed. Token stored in system keychain via keytar; falls back to AES-256-GCM encrypted file in ~/.epam/credentials.json if keytar is unavailable.

epam keys Manage BYOK API keys per provider AUTH 
epam keys set <provider> <key>    # anthropic | openai | gemini
epam keys get <provider>           # Show masked key
epam keys remove <provider>        # Delete stored key
epam keys list                      # List providers with stored keys
epam config Show or edit CLI configuration CORE 
epam config show          # Print resolved config (all sources merged)
epam config get <key>     # Get a specific config value
epam config set <key> <value>  # Set a config value in .epam/settings.json

Config priority: CLI flags > EPAM_* env vars > .epam/settings.json (project) > ~/.epam/config.json (global) > defaults.

epam models List available models and their pricing CORE 
epam models                 # List all models with pricing info
epam models --provider anthropic  # Filter by provider
epam history List or view past sessions CORE 
epam history                      # List recent sessions
epam history show <session-id>    # Print full session transcript
epam context                      # Show/manage context.md for current project

Sessions are persisted in .epam/sessions/ within the project root. Use /resume inside the REPL to restore a previous session.

epam doctor Check runtime dependencies and configuration health CORE 
epam doctor

Checks: Node.js version (≥20), keytar availability, configuration validity, backend connectivity (pro/enterprise), API key presence, tool permissions.

epam estimate Forecast AI cost, tokens, minutes, and turns per story ★ NOVEL
Not found on other AI CLI tools
epam estimate [--phase <id>] [--apply] [--refine] [--skip-cpa] [--strict]
              [--json] [--dry-run] [--reconcile]

# Dry run — see estimates without writing anything
epam estimate --phase finops

# Full pass: formula + CPA, write results to prd.json
epam estimate --phase finops --apply

# Formula only (no LLM)
epam estimate --skip-cpa

# Calibrate model constants from historical actuals
epam estimate --refine

# JSON output for scripting
epam estimate --json | jq '.[0]'

# Compare prior CPA estimates against actuals (post-phase retrospective)
epam estimate --reconcile
FlagEffect
--phase <id>Scope to one phase (e.g. finops, health_check)
--applyWrite blended estimates to prd.json (default: dry-run)
--refineCalibrate formula constants from phase-cost.jsonl actuals
--skip-cpaSkip the CPA LLM pass (formula only, very fast)
--strictHalt on CPA REVIEW gates (default: halt only on BLOCK)
--jsonOutput full results as JSON array
--dry-runRun inference but skip all writes
--reconcilePost-phase: compare CPA estimates vs. actuals

See CPA / Estimation details for the full model documentation.

epam orchestrate Run multi-agent orchestration for a project phase ★ NOVEL
Not found on other AI CLI tools
epam orchestrate [--phase <id>] [--mode bash|hybrid] [--dry-run]
                 [--skip-cleanup] [--skip-cpa] [--strict-cpa]

# Run the finops phase with bash (sequential) mode
epam orchestrate --phase finops

# Hybrid mode: parallel git worktrees for independent stories
epam orchestrate --phase finops --mode hybrid

# Preview what would run without executing agents
epam orchestrate --phase finops --dry-run

# Skip CPA gate (override BLOCK/REVIEW — use with caution)
epam orchestrate --phase finops --skip-cpa

# Keep worktrees after execution for debugging
epam orchestrate --phase finops --skip-cleanup

Delegates to run-agent-orchestration.sh which executes the full 8-step pipeline: infra gate → CPA pre-pass → skill assessment → agent execution → code review → unit tests → phase gate → post-assessment. See the pipeline diagram for the full flow.

Exit CodeMeaning
0Phase completed successfully
1General failure (gate failed, test failed, etc.)
2Escalated — max review iterations exhausted, needs human review
3CPA gate BLOCK — flagged stories must be resolved before proceeding
epam phase Phase control plane: estimate + approve + orchestrate in one command ★ NOVEL
Not found on other AI CLI tools
epam phase run    --phase <id> [--model <name>] [--dry-run] [--skip-cpa]
                              [--strict-cpa] [--require-approval]

epam phase approve --phase <id>

# Full phase workflow with human approval gate:
epam estimate --phase finops --apply        # 1. Generate estimates
epam phase approve --phase finops           # 2. Human reviews estimates + approves
epam phase run --phase finops               # 3. Execute (will use prior approval)

# Require approval before execution (CI/CD gate):
epam phase run --phase finops --require-approval

# Dry run — see what would happen
epam phase run --phase finops --dry-run

phase approve

Shows a preview of the phase: story list, per-story estimates (minutes, cost, tokens, turns), and totals. Prompts Approve this phase for execution? [y/N]. On confirmation, writes a signed approval record to orchestrations/logs/phase-approvals.jsonl containing the approver identity (from EPAM_APPROVER, GIT_AUTHOR_EMAIL, USER, or OS username), timestamp, story fingerprint (SHA-256), and estimates snapshot.

The approval is invalidated if prd.json changes after approval (story fingerprint mismatch). Re-approve after any estimate update.

phase run

Runs the full pipeline in sequence: epam estimate (with optional CPA) → epam orchestrate. Logs each stage to timestamped files in orchestrations/logs/cli-phase-runner/<phase>-<timestamp>/. Emits structured phase_start, phase_complete, and phase_fail events to stdout as JSON (useful for CI/CD pipelines).

Model selection with EPAM provider: Use --model to override the resolved model. The model must exist in configuration.aiRuntime.allowedModels in prd.json. Model selection source is logged: story → phase config → project config → effort tier.

Novel Slash Commands ★

These commands are not standard on other AI CLI tools. They are specific to EPAM CLI's enterprise/orchestration feature set.

/cost — Live session cost tracking with per-model pricing breakdown and budget guardrail status. Shows input tokens, output tokens, input cost, output cost, total cost, warning threshold %, and hard limit remaining. Most AI CLIs show usage at the end of a session; /cost shows it any time mid-session.

/chain — Live view and runtime control of the LLM provider failover chain. Shows circuit breaker state per slot (healthy/degraded/down/unavailable), active provider, cooldown timers, and last error. Can redefine the priority chain or reset all circuit breakers without restarting the session.

# View chain status
/chain

LLM Provider Chain:

  1 ✓ anthropic/claude-sonnet-4-6   healthy  [active]
  2 ~ openai/gpt-4o                 degraded  retry in 47s  (rate limit)
  3 ○ gemini/gemini-1.5-pro         unavailable  (no API key)

# Redefine the chain
/chain set anthropic/claude-opus-4-6 openai/gpt-4o

# Reset all circuit breakers
/chain reset

/rewind — Removes the last complete turn (user message + assistant response) from the conversation history. Shows a preview of what will be removed and asks for confirmation. Useful when a response went in the wrong direction and you want to rephrase your prompt.

/plan [show|create|branch] — Structured plan mode. /plan show displays the current strategy and branch count. /plan create starts a new step-by-step plan. /plan branch shows available branching options. Alias: /planning.

/agent [switch|show|add|remove|reset] — Named agent persona management. /agent lists all available agents (built-in, session, and orchestration personas). /agent switch <name> activates an agent for the next message. /agent add <name> <prompt> saves a custom session agent. /agent reset restores the default system prompt. Orchestration agents from orchestrations/agents/profiles.json are visible but cannot be removed. Alias: /agents.

/stash [save|list|pop|merge|drop] — Private context session stash. /stash save [name] snapshots the current conversation to ~/.epam/stash/. /stash list shows all your stashes with message counts and timestamps. /stash pop <name|id> restores a stash, replacing the current context. /stash merge <name|id> appends stash messages into the current context. /stash drop <name|id> deletes a stash permanently. Stashes are private (keyed by username) and survive session restarts.

Configuration Files

EPAM CLI uses a layered config system. Lower items are overridden by higher items:

PrioritySourceScope
1 (highest)CLI flags (--model, --provider)Command only
2EPAM_* environment variablesProcess
3.epam/settings.jsonProject (committed)
4~/.epam/config.jsonGlobal (user)
5 (lowest)Built-in defaultsAlways

Key settings.json fields

{
  "provider": "anthropic",           // anthropic | openai | gemini | epam
  "model":    "claude-sonnet-4-6",
  "backendUrl": "http://localhost:8080",

  // Provider failover chain (up to 5 slots)
  "llmChain": [
    { "provider": "anthropic", "model": "claude-sonnet-4-6" },
    { "provider": "openai",    "model": "gpt-4o" }
  ],

  // Budget guardrails
  "budget": {
    "warningAt":  1.00,   // warn when session cost exceeds $1.00
    "hardLimitAt": 5.00   // halt when session cost exceeds $5.00
  },

  "contextFile":    ".epam/context.md",
  "systemPromptFile": ".epam/system.md",
  "maxIterations":  50,
  "autoCompressAt": 80000,   // tokens before auto-compacting conversation
  "maxOutputTokens": 8192,
  "tools": {
    "dangerousSkipApproval": false   // set true for CI/CD automation
  }
}

Provider Failover Chain ★

Not found on other AI CLI tools

EPAM CLI implements a transparent LLM provider failover chain — if one provider fails (rate limit, timeout, server error), it automatically switches to the next slot without interrupting the conversation. The chat context is seamlessly transported to the new provider.

Configuring the chain

# In .epam/settings.json
{
  "llmChain": [
    { "provider": "anthropic", "model": "claude-sonnet-4-6",  "label": "primary" },
    { "provider": "openai",    "model": "gpt-4o",             "label": "fallback" },
    { "provider": "anthropic", "model": "claude-haiku-4-5-20251001", "label": "economy" }
  ]
}

Failover policy

HTTP StatusAction
429 (rate limit)Immediate failover to next slot
503 / 529 (overloaded)Immediate failover
5xx / timeoutRetry once on same slot, then failover
400 / 401 / 403Fatal — do not retry or failover (config error)

A failed slot enters circuit breaker cooldown (default 60s) before being retried. A slot is marked unavailable if no API key is configured for it. When a failover fires, a warning line is printed to stderr: 

⚠  Failover: anthropic/claude-sonnet-4-6 → rate limit → switching to openai/gpt-4o

Use /chain inside the REPL to inspect live status and /chain reset to clear cooldowns.
