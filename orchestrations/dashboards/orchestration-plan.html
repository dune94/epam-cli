Phases 1–4, 7 Mode
bash
Health checks, FinOps, agent intelligence, team features, and RAG POC — sequential execution, no coordination needed

hybrid
Multi-agent and enterprise phases benefit from parallel worktree execution, skill assessment, KB.md feedback, and coordination bus

TypeScript strict

Node 20 LTS — required for ESM compatibility, native fetch, and modern APIs. Ubuntu WSL2 + Docker Compose
Bundler
tsup (CJS)
tsup bundles to CommonJS for maximum compatibility. Output at dist/epam.js with shebang entry point
Testing
vitest
vitest for unit + integration tests. 42/42 tests passing. Coverage tracked per phase gate
Multi-LLM
Claude + OpenCode + Codex + Codemie

Skill Assessment
Step 0.5 enabled
Pre-phase agent profile gap analysis for hybrid phases (5–6). Required skills in technicalNotes.requiredSkills per story
Knowledge Base
KB.md enabled
Learnings written on first-success and retries; injected per agent role. Shared across all phases via orchestrations/agents/KB.md

Pipeline Steps (All Phases)

Phases 1–4 and 7 (health_check, finops, agent_intelligence, team_features, rag_poc) use bash mode — implementation stories run sequentially in main lane. Hybrid phases (5–6) enable parallel worktree execution + coordination bus.

Domain engineer (typescript-engineer, agent-systems-engineer, billing-engineer) validates output against acceptance criteria post-implementation
Steps 4.2–4.4
QA Testing Gates
3 cascading phases: Phase A (sast-sentinel + spec-validator) → Phase B (review-ranger + mutant-hunter) → Phase C (fuzz-weaver + perf-sentinel). Each phase runs agents in parallel. Blocks on any blocker finding. Bypass: SKIP_TESTING_GATES=true
Step 5
Phase Gate
Cost variance check: warn at +30%, block at +150%. Results written to phase-gates.jsonl. Unblocks next phase.

QA Gate Agents (Steps 4.2–4.4)

These agents run automatically inside run_testing_gates() between code review (Step 4) and unit tests (Step 4.5). They are pipeline infrastructure — not assigned to stories. Bypass all with SKIP_TESTING_GATES=true.

Agent Gate Phase Trigger Runs With Verdict Policy
test-coordinator-agent Coordinator Governs all 3 phases — Aggregates agent verdicts, logs to testing-gates.jsonl
sast-sentinel Phase A (4.2) Always (code phases) spec-validator Any blocker → FAIL
spec-validator Phase A (4.2) Always (code phases) sast-sentinel Compliance < 60% → FAIL, 60–79% → WARN
review-ranger Phase B (4.3) Only if Phase A passed mutant-hunter Any blocker → FAIL
mutant-hunter Phase B (4.3) Only if Phase A passed review-ranger Score < 50% → FAIL, 50–69% → WARN
fuzz-weaver Phase C (4.4) Only if A+B passed perf-sentinel Any vulnerability → FAIL, gaps > 30% → WARN
perf-sentinel Phase C (4.4) Only if A+B passed fuzz-weaver Any blocker or high-impact major → FAIL
